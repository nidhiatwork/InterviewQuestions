Code Python — synced to animation

def numIslands(grid):
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != '1':
            return                          # off-grid or not land
        grid[r][c] = '0'                   # sink: mark visited
        dfs(r+1, c); dfs(r-1, c); dfs(r, c+1); dfs(r, c-1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1                  # found a new island!
                dfs(r, c)                   # sink the whole thing
    return count

Plain English What you just watched

The grid is a map. '1' = land, '0' = water. An island is a clump of land cells that touch each other horizontally or vertically (diagonals don't count). Your job: count the islands.

The trick: when you find a land cell, "sink" the entire island it belongs to so you never count it again. Then keep walking. Every time you bump into land that hasn't been sunk yet → that's one more island.

How to sink an island — DFS flood-fill (the orange cell in the animation):

  • Standing on a land cell, overwrite it to '0'.
  • Recursively do the same to its 4 neighbors (up, down, left, right).
  • Each recursive call either falls off the grid, hits water, or hits land it then sinks too — so the recursion naturally stops at the island's edge.
  • When DFS returns, the whole connected island is now '0' (the faded blue cells).

The outer cursor (red outline) just scans the grid left-to-right, top-to-bottom. It triggers a DFS only when it sees a fresh '1' — which is exactly once per island, hence one count++ per island.

Why this is O(m × n): each cell is touched at most twice — once by the outer cursor, once by a DFS visit. After DFS sinks a cell, the outer cursor will skip it on later passes (it's now '0').

The single mental picture to remember: 🏝️ "Walk the map. Every time you step on a new island, plant a flag (++count) and let the tide swallow it whole (DFS). Keep walking."

3 traps What interviewers check

  • Diagonals don't connect. The DFS recurses on exactly 4 directions — up, down, left, right. State this out loud before coding; it's the most common silent assumption.
  • In-place mutation vs visited set. Overwriting '1' → '0' is O(1) extra space but mutates the input. If the interviewer says "don't touch the input," switch to a visited set — same algorithm, slightly more memory.
  • Recursion depth. On a 300×300 all-land grid, DFS recursion is 90,000 deep — risky in Python. Mention you'd switch to BFS (queue) as the safer variant for very large grids.
