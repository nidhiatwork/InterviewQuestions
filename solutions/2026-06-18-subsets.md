# LeetCode #78 - Subsets

**Data structure:** Backtracking  
**Difficulty:** Medium  
**URL:** https://leetcode.com/problems/subsets/

## Problem

Given an integer array `nums` of **unique** elements, return all possible subsets (the power set).

The solution set must not contain duplicate subsets. Return them in any order.

## Examples

```text
Input: nums = [1,2,3]
Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]
```

```text
Input: nums = [0]
Output: [[],[0]]
```

## Constraints

- `1 <= nums.length <= 10`
- `-10 <= nums[i] <= 10`
- All numbers in `nums` are unique.

## Approach

Backtracking: at each index, decide to include or skip that element, recording a subset at every node of the decision tree.

A recursive helper takes a `start` index and the current partial subset:

1. Record a **copy** of the current subset (every recursion node is a valid subset, including the empty one at the start).
2. For each index `i` from `start` to the end:
   - Choose `nums[i]` (append).
   - Recurse with `start = i + 1` so earlier elements are never reused and permutations of the same set are never produced.
   - Un-choose (pop) to backtrack and try the next candidate.

Always advancing the start index generates each subset exactly once in increasing-index order, so there are no duplicates.

**Complexity**

- Time: `O(n · 2^n)` — `2^n` subsets, each up to length `n` to copy
- Space: `O(n)` recursion depth (excluding the output)

## Python solution

```python
class Solution:
    def subsets(self, nums):
        result = []

        def backtrack(start, path):
            result.append(path[:])           # record current subset
            for i in range(start, len(nums)):
                path.append(nums[i])         # choose
                backtrack(i + 1, path)       # explore (i+1 avoids reuse/dups)
                path.pop()                   # un-choose

        backtrack(0, [])
        return result
```

## unittest test cases

```python
import unittest


def normalize(subsets):
    return sorted([sorted(s) for s in subsets])


class TestSubsets(unittest.TestCase):
    def setUp(self):
        self.sol = Solution()

    def test_example_1(self):
        result = self.sol.subsets([1, 2, 3])
        expected = [[], [1], [2], [3], [1, 2], [1, 3], [2, 3], [1, 2, 3]]
        self.assertEqual(normalize(result), normalize(expected))

    def test_single_element(self):
        result = self.sol.subsets([0])
        self.assertEqual(normalize(result), normalize([[], [0]]))

    def test_count_is_power_of_two(self):
        nums = [1, 2, 3, 4]
        self.assertEqual(len(self.sol.subsets(nums)), 2 ** len(nums))

    def test_no_duplicate_subsets(self):
        result = self.sol.subsets([5, 6, 7])
        as_tuples = [tuple(sorted(s)) for s in result]
        self.assertEqual(len(as_tuples), len(set(as_tuples)))
```

## Interview tips

- Record a subset at **every** node, not only the leaves — the empty set and all partial sets are valid.
- Passing `start = i + 1` (not `0`) is what prevents duplicates and permutations.
- Append a **copy** (`path[:]`) — appending the live list would be mutated later by backtracking.
- Two clean alternatives: **iterative** — start with `[[]]` and add each num to every existing subset; **bitmask** — iterate masks `0..2^n - 1` and include `nums[j]` when bit `j` is set.
- The bitmask version is a great one-liner to mention for small `n`.
