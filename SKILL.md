# /dsa-daily — Daily DSA Interview Question Picker

Picks one **Medium-difficulty LeetCode problem** every day at 10:03 AM IST, rotating through 14 data-structure categories in round-robin order. Generates a Python solution with test cases as a markdown file, posts a `/msg-me` notification with a clickable link, and surfaces today's pick in the PulseIQ dashboard.

## Trigger

Use when the user says `/dsa-daily`, `/dsa-daily pick`, `/dsa-daily list`, `/dsa-daily show`, `/dsa-daily start`, or any cron-fired prompt that begins with `Pick today's daily DSA interview question`.

## Modes

| User says | Mode | What happens |
|-----------|------|--------------|
| `/dsa-daily` or `/dsa-daily start` | **Setup** | Creates the daily 10:03 AM IST cron and runs Pick once immediately |
| `/dsa-daily pick` (or cron-fired prompt) | **Pick** | Selects today's question, generates solution, posts via `/msg-me` |
| `/dsa-daily list` | **List** | Shows all previously picked questions grouped by data structure |
| `/dsa-daily show` | **Show** | Shows today's question (if already picked) or yesterday's |
| `/dsa-daily reset-rotation` | **Reset** | Resets `rotationIndex` to 0 |

## Mode: Pick (`/dsa-daily pick` or cron-fired)

### Step 1 — Read state

Open `state.json`. Determine today's `dataStructure = rotation[rotationIndex]`. Today is `lastPickedDate` (UTC date, formatted `YYYY-MM-DD`).

If `lastPickedDate == today` and `askedQuestions[-1].date == today`, the cron already fired today — print `Already picked for today: <title>` and exit silently.

**Pre-flight straggler check:** before doing anything else, `cd` to the skill root and run `git status`. If there are untracked or modified files from a prior run (e.g., a previous cron crashed mid-flight), commit + push them now with message `Sync pending /dsa-daily artifacts` so we start from a clean tree.

### Step 2 — Pick a fresh problem from LeetCode

**Primary path (live LeetCode):**
1. Open `mcp__playwright__browser_close` first to reset any stale browser, then `mcp__playwright__browser_navigate` to `https://leetcode.com/problemset/?difficulty=MEDIUM&topicSlugs=<slug>` where `<slug>` is the LeetCode topic slug for today's data structure (table below).
2. Wait for problem list to render.
3. Take a `mcp__playwright__browser_snapshot` and extract the first 20 problem rows. Each row has the LeetCode #, title, slug (in the link href), difficulty.
4. Filter to `difficulty == "Medium"` and skip any whose `leetcodeId` already appears in `askedQuestions[]`.
5. Pick the first remaining problem. Record `leetcodeId`, `title`, `slug`, `url = https://leetcode.com/problems/<slug>/`.

**Fallback path (curated seed list):**
If Playwright fails (page won't load, login wall, etc.), read `state/seed-problems.json` and pick the first `dataStructure`-matching problem whose `leetcodeId` is not in `askedQuestions[]`. The seed list is pre-vetted Microsoft-interview-relevant Medium problems (~10 per DS).

### Step 3 — Generate the single playground file

Compose ONE self-contained Python file at:
```
C:\Users\bhushanidhi\.copilot\skills\dsa-daily\playground\<YYYY-MM-DD>-<slug>.py
```

This file is BOTH the boilerplate the user practises on AND the reference solution they peek at when stuck. There is no separate `solutions/` markdown.

Structure (top to bottom):

1. **Module docstring** — title with LeetCode # + DS + difficulty + URL, problem statement, 2-3 examples, constraints, and the run command (`python <file>.py -v`).
2. **Imports** — only what tests/helpers need (`unittest`, `collections.deque`, etc.). NO `from typing import ...`.
3. **Boilerplate class** — `Solution` (or `MyCircularQueue`, `Trie`, etc.) with **every method body** replaced by `raise NotImplementedError("Implement <method>")`. **Strip ALL type annotations** — `def threeSum(self, nums):` not `def threeSum(self, nums: List[int]) -> List[List[int]]:`. Helper node classes (`ListNode`, `TreeNode`) also annotation-free: `def __init__(self, val=0, next=None):`.
4. **Helper functions** the tests need (e.g. `list_to_linked`, `build_tree`) — inlined, no annotations.
5. **`unittest.TestCase` block** — at least 5 cases covering provided examples + edges (empty/single/duplicates/large/tricky boundaries).
6. **`if __name__ == "__main__": unittest.main()`**
7. **Reference solution section at the very bottom** — a single triple-quoted string assigned to `REFERENCE`, preceded by a banner:
   ```
   # ============================================================================
   # REFERENCE SOLUTION & NOTES  — peek only if stuck
   # ============================================================================

   REFERENCE = """
   Approach
   --------
   <short narrative>

   Complexity
   ----------
   - Time:  ...
   - Space: ...

   Python solution
   ---------------
   class Solution:
       def methodName(self, args):
           <full canonical body>

   Interview tips
   --------------
   - ...
   """
   ```
   The reference goes inside a string (NOT executable) so the file still imports the stubbed `Solution`. The user reads the string with their editor or `print(REFERENCE)` in a REPL when they want a peek.

   **Non-DP framing rule (preference):** Nidhi prefers non-DP solutions for interview practice. When the rotation lands on `Dynamic Programming` (or any problem that's canonically solved with DP), frame the reference solution in non-DP terms — *state machine / streaming / rolling variables / greedy / math*. Use two rolling variables instead of a `dp[]` array, and never use the words "dp", "memoization", "recurrence", or "Dynamic Programming" in the Approach narrative or Interview tips. The algorithm cost can be identical to space-optimized DP — only the framing changes. Title line for these picks: `(Array · Medium)` / `(<DS> · Medium)` rather than `(Dynamic Programming · Medium)`. See `playground/2026-05-18-house-robber.py` for the canonical example.

Purpose: a single file the user opens, codes against, and runs. The reference at the bottom replaces what used to be the separate `solutions/<date>-<slug>.md`. See `playground/README.md` for the workflow.

### Step 4 — Post to Teams

The goal is **one clickable hyperlink in Teams** that opens the markdown file in the user's browser (where a browser markdown extension renders it). Two sub-steps:

**4a — Upload the .md to SharePoint** (silent — to get a real URL) via `mcp__teams__SendFileToChannel`:
- `fileName`: `<YYYY-MM-DD>-<slug>.md`
- `fileContentBase64`: base64 of the solution markdown. Encode via PowerShell:
  ```powershell
  $bytes = [System.IO.File]::ReadAllBytes('<solutionPath>')
  $b64 = [System.Convert]::ToBase64String($bytes)
  $b64 | Set-Content -Path "$env:TEMP\dsa-b64.txt" -NoNewline
  ```
  Then read `$env:TEMP\dsa-b64.txt` and pass to the tool.
- `message`: leave empty or omit (we don't want the file-card noise — only the hyperlink in 4b should be visible).

**Capture** the returned `fileUrl` (a SharePoint URL ending in `/<filename>.md`). Save it in state as `sharePointUrl`. Note: SendFileToChannel itself posts a file-card to the channel — that's acceptable, but the canonical user-facing link comes in 4b.

**4b — Announcement message with browser-open link** via `mcp__teams__SendMessageToChannel`:
- `contentType`: `html`
- `mentions`: `[{"displayName":"Nidhi Bhushan","id":"143b173b-ab84-48eb-9e65-7d28650e4349","type":"user"}]`
- Append `?web=1` to the SharePoint URL — this forces SharePoint to serve the file inline (so the browser markdown extension can render it) instead of triggering a download.
- `content`:
  ```html
  <b>[Sent from Nidhi's Clawpilot]</b> - @Nidhi Bhushan<br>
  <b>[Cron: /dsa-daily]</b><br>
  <b>📚 Today's DSA Interview Question — <i>{dataStructure}</i></b><br><br>
  <b>LeetCode #{leetcodeId}: <a href="{url}">{title}</a></b> · {difficulty}<br>
  👉 <a href="{sharePointUrl}?web=1">Open <b>{date}-{slug}.md</b> in browser</a>
  ```
- The `?web=1` query parameter is the SharePoint convention for "render inline, don't download." The user's browser markdown extension takes over from there.

### Step 5 — Update state

- Append the new entry to `askedQuestions[]`
- Set `lastPicked` and `lastPickedDate` to today's pick
- Increment `rotationIndex = (rotationIndex + 1) % rotation.length`
- Write `state.json` back
